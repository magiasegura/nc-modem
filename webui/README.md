# modemui

Лёгкий веб-интерфейс расширенных функций LTE-модема для роутеров **Keenetic** с **Entware** —
то, чего нет в штатном интерфейсе Keenetic.

Один статический бинарь (~350 КБ), **ноль внешних зависимостей**: только стандартная библиотека
Rust, фронтенд вшит в бинарь. В `/opt` ничего не доустанавливается.

## Возможности

| Раздел | Что делает |
|---|---|
| Сигнал | RSRP / RSRQ / SINR, текущая сота, оператор, статус регистрации, график за последний час |
| Фиксация соты | Установка и снятие `earfcn_lock` / `pci_lock`, перезагрузка модема |
| Соседние соты | Список соседей с RSRP/RSRQ и кнопкой «зафиксировать эту» |
| Бэнды | Чтение и запись маски диапазонов |
| AT-консоль | Произвольные AT-команды |

Набор AT-команд у прошивок T77W968 отличается, поэтому при старте выполняется **проба
возможностей**: каждый кандидат отправляется модему, запоминается ответивший. Разделы, которые
конкретная прошивка не поддерживает, в интерфейсе просто не появятся.

## Установка

```sh
curl -fsSL https://raw.githubusercontent.com/magiasegura/nc-modem/main/setup.sh | sh
```

Установщик определяет архитектуру, скачивает нужный бинарь, кладёт `/opt/bin/modemui`,
создаёт конфиг `/opt/etc/modemui.conf` и сервис `/opt/etc/init.d/S99modemui`.

## Удаление

```sh
curl -fsSL https://raw.githubusercontent.com/magiasegura/nc-modem/main/uninstall.sh | sh -s -- --unlock
```

`--unlock` снимает фиксацию соты перед удалением. Это важно: фиксация хранится в NV-памяти
модема и удалением файлов не снимается, а после удаления снять её будет нечем. При активной
фиксации скрипт требует явного выбора — `--unlock` или `--keep-lock`. Добавьте `--purge`,
чтобы удалить и конфиг.

## Настройка

`/opt/etc/modemui.conf`, после правки — `/opt/etc/init.d/S99modemui restart`:

```sh
LISTEN="0.0.0.0:1010"   # адрес и порт
USER="admin"            # логин Basic-аутентификации
PASS="секрет"           # пароль
IFACE=""                # UsbQmi0 / UsbLte0, пусто = автоопределение
DEV=""                  # /dev/ttyACM0, пусто = автоопределение
```

**Без `USER`/`PASS` интерфейс открыт всем в локальной сети без пароля.** Сервис при старте
пишет об этом предупреждение в лог.

## Запуск вручную

```
modemui [флаги]

  -l, --listen <addr:port>  адрес прослушивания (по умолчанию 0.0.0.0:1010)
  -i, --iface <IF>          интерфейс ndmc (UsbQmi0/UsbLte0), иначе автоопределение
  -d, --dev <path>          AT-порт напрямую (/dev/ttyACM0), иначе автоопределение
  -u, --user <name>         логин Basic-аутентификации
  -p, --pass <pass>         пароль Basic-аутентификации
      --demo                фиктивный модем: посмотреть интерфейс без железа
  -h, --help                справка
```

Посмотреть интерфейс на своей машине, без роутера и модема:

```sh
cargo run --release -- --demo -l 127.0.0.1:8080
```

## Архитектура

```
src/at.rs      транспорт AT: ndmc / прямой /dev/ttyACM* / mock; разбор ответов
src/modem.rs   EFS-фиксация, LE-конверсия, парсеры метрик и соседей, история
src/http.rs    HTTP/1.1 на TcpListener, Basic-auth, лимиты
src/json.rs    сериализация JSON и разбор form-urlencoded
src/api.rs     роуты, статика через include_str!
web/           index.html, app.js, style.css — вшиваются в бинарь
```

**AT-порт модема один и не реентерабельный.** Весь доступ идёт через `Mutex<AtPort>` внутри
`Modem`, поэтому UI и параллельные запросы не могут перемешать команды. Метрики собирает
фоновый поток раз в 5 секунд, а HTTP-запросы отдают закэшированное значение — иначе каждый
тик интерфейса дёргал бы порт и блокировал запись.

## API

| Метод | Путь | Назначение |
|---|---|---|
| GET | `/api/status` | транспорт, возможности, состояние фиксации, последний сигнал |
| GET | `/api/history` | точки графика |
| GET | `/api/bands` | текущая маска бэндов |
| POST | `/api/lock/earfcn` | `earfcn=<N>` |
| POST | `/api/lock/pci` | `earfcn=<N>&pci=<N>` |
| POST | `/api/unlock` | снять обе фиксации |
| POST | `/api/reset` | `at+cfun=1,1` |
| POST | `/api/scan` | список соседних сот |
| POST | `/api/bands` | `mask=<hex>` |
| POST | `/api/at` | `cmd=<AT-команда>` |

Тела запросов — `application/x-www-form-urlencoded`, ответы — JSON. Ошибки: `{"error": "…"}`
с кодом 400.

## Сборка

Тесты и проверки на хосте:

```sh
cargo fmt --check
cargo clippy --all-targets -- -D warnings
cargo test
```

Кросс-сборка под роутеры идёт в GitHub Actions (`.github/workflows/build.yml`):

- `aarch64-unknown-linux-musl`, `armv7-unknown-linux-musleabihf` — Tier 2, стабильный Rust;
- `mipsel-unknown-linux-musl`, `mips-unknown-linux-musl` — **Tier 3**: готового `std` нет,
  нужен nightly с `-Z build-std` и musl-кросслинкер. Эта джоба помечена `continue-on-error`,
  чтобы релиз для aarch64/armv7 выходил, даже если mips-сборка не сложится.

### Закреплённые версии

В `env` workflow закреплены две вещи, и это осознанно:

| Переменная | Зачем |
|---|---|
| `RUST_NIGHTLY` | Tier 3 требует nightly, а нестабильные флаги в нём меняются без предупреждения — `-Z build-std-features=panic_immediate_abort` уже сломался на ровном месте. |
| `MUSL_CROSS_RELEASE` | Скачивание по ссылке `latest` означает, что переименование ассета молча уронит сборку. |

Обновлять их следует вручную и по одной, чтобы при поломке было понятно, что именно её вызвало.
Проверить, что новая дата nightly существует и содержит `rust-src`:

```sh
curl -sL https://static.rust-lang.org/dist/<ГГГГ-ММ-ДД>/channel-rust-nightly.toml | grep -c pkg.rust-src
```

### Float-ABI на mips

Таргеты `mips*-unknown-linux-musl` в Rust собираются с `+soft-float` — проверяется так:

```sh
rustc +nightly -Z unstable-options --print target-spec-json --target mipsel-unknown-linux-musl
```

Поэтому берутся тулчейны с суффиксом **`sf`** (`mipsel-unknown-linux-muslsf`), а не обычные:
обычные собраны hard-float, и при смешивании float-аргументы передаются через разные регистры.
Линкер выдаёт лишь предупреждение, а ломается всё молча в рантайме, поэтому после сборки
ABI проверяется через `readelf -A`.

## Предупреждения

- Фиксация несущей/сектора пишет в NV-память модема. Если зафиксированной соты нет в эфире,
  модем потеряет регистрацию — снимается кнопкой «Снять фиксацию», для чего нужен доступ
  к роутеру по LAN, а не через сам модем.
- Фиксация EARFCN и фиксация PCI взаимоисключающие: включение одной снимает другую.
- Неверная маска бэндов может лишить модем связи.
- AT-консоль позволяет отправить что угодно, включая `at^efs`, который пишет в NV напрямую
  и может окирпичить модем.

Подробнее о методе фиксации — в [README корня репозитория](../README.md).
