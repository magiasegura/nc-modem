# modem_keenetic

Расширенные функции LTE-модема на роутерах **Keenetic** с **Entware** — то, чего нет
в штатном интерфейсе Keenetic.

Разработано для **Foxconn T77W968 / Dell DW5821e**. Тот же метод известен для Fibocom LT4220
и, по сообщениям, Sierra EM7455 — но на них не проверялось.

Две части:

| | Что это |
|---|---|
| **[`lockband`](lockband)** | CLI-скрипт: фиксация несущей/сектора из командной строки. Чистый POSIX sh, без зависимостей. |
| **[`webui/`](webui/README.md)** | Веб-интерфейс: фиксация, мониторинг сигнала с графиком, скан соседних сот, бэнды, AT-консоль. Один бинарь ~350 КБ. |

Установка обеих частей одной командой:

```sh
curl -fsSL https://raw.githubusercontent.com/OWNER/REPO/main/setup.sh | sh
```

# lockband

Фиксация несущей (EARFCN) и/или сектора (PCI) из командной строки.

## Как это работает

У Keenetic нет штатного способа зафиксировать базовую станцию для этого модема, а сам
T77W968 не понимает привычных `AT!BAND` / `at@sic:freq_lock`. Зато он поддерживает
команду `at^efs`, которой можно писать NV/EFS-файлы модема напрямую:

| Файл | Размер | Содержимое |
|---|---|---|
| `/nv/item_files/modem/lte/rrc/csp/earfcn_lock` | 2 байта | EARFCN, little-endian |
| `/nv/item_files/modem/lte/rrc/csp/pci_lock` | 4 байта | EARFCN LE + PCI LE |

Например, EARFCN 2850 = `0x0B22` → байты `22,0b`; PCI 57 = `0x0039` → `39,00`:

```
at^efs="/nv/item_files/modem/lte/rrc/csp/pci_lock",4,"22,0b,39,00"
at+cfun=1,1
```

Скрипт принимает числа в десятичном виде, сам делает конверсию, отправляет команды
и проверяет, что записалось именно то, что нужно.

Источник метода: [4PDA, тема 994474](https://4pda.to/forum/index.php?showtopic=994474&st=13300#entry108880980).

## Требования

- Keenetic с установленным Entware (opkg) и доступом по SSH;
- KeeneticOS 3.9+ (для отправки AT-команд через `ndmc`) **или** доступный AT-порт `/dev/ttyACM*`;
- только busybox — внешних пакетов не нужно.

## Установка

```sh
# на роутере
curl -fsSL <url>/lockband -o /opt/bin/lockband
chmod +x /opt/bin/lockband
```

Или просто скопировать файл `lockband` в `/opt/bin/` (обязательно с LF-переводами строк).

## Использование

```
lockband status                    показать текущие earfcn_lock / pci_lock
lockband cell                      показать текущую соту (EARFCN/PCI), best-effort
lockband lock-earfcn <EARFCN>      зафиксировать только несущую
lockband lock-pci <EARFCN> <PCI>   зафиксировать несущую + сектор
lockband unlock                    снять обе фиксации
lockband at '<AT-команда>'         отправить произвольную AT-команду (отладка)
```

Флаги:

| Флаг | Что делает |
|---|---|
| `-n`, `--dry-run` | только показать AT-команды, ничего не отправлять |
| `-i`, `--iface <IF>` | интерфейс ndmc (`UsbQmi0` / `UsbLte0`), иначе автоопределение |
| `-d`, `--dev <path>` | AT-порт напрямую (`/dev/ttyACM0` …), иначе автоопределение |
| `-y`, `--yes` | не спрашивать подтверждение перед `at+cfun=1,1` |
| `-v`, `--verbose` | показывать сырой обмен с модемом |

### Пример

```sh
# 1. посмотреть, к чему подключены сейчас
lockband cell
#   EARFCN(DL/UL): 1575/19575
#   eNB ID(PCI): 25733-2(161)
#   Похоже: EARFCN=1575 PCI=161
#     lockband lock-pci 1575 161

# 2. зафиксировать несущую
lockband lock-earfcn 1575

# 3. проверить
lockband status
#   earfcn_lock : 1575	(27,06)
#   pci_lock    : not set

# 4. вернуть как было
lockband unlock
```

`lock-earfcn` и `lock-pci` **взаимоисключающие**: при установке одного второй автоматически
удаляется, иначе в модеме возникает конфликт приоритетов.

### Как узнать EARFCN и PCI

- `lockband cell` — если прошивка модема отвечает на диагностические команды;
- приложение **netmonitor** на смартфоне;
- карта **cellmapper.net**;
- приложение **NSG** (нужен root).

## Транспорт

Скрипт сам выбирает, как разговаривать с модемом:

1. **ndmc** — `ndmc -c 'interface UsbQmi0 tty send …'`, штатный путь Keenetic.
   После каждой записи делается контрольное чтение: CLI Keenetic может исказить кавычки
   в `at^efs="…"`, и тогда скрипт автоматически переключается на способ 2.
2. **Прямой AT-порт** — перебор `/dev/ttyACM0..3` и `/dev/ttyUSB0..3`, первый ответивший
   на `ATE0` кэшируется. Порт может быть занят прошивкой роутера.

Если не работает ни то, ни другое — скрипт выйдет с кодом 2 и подсказкой. Можно задать
транспорт вручную: `-i UsbQmi0` или `-d /dev/ttyACM0`.

## Предупреждения

- **Записи носят для сети рекомендательный характер.** Если оборудование БС решит, что вам
  «будет лучше» с другой станцией (например, по Timing Advance), вас всё равно переключат.
- **Фиксация на несуществующий сектор = потеря регистрации.** Модем будет молча ждать сигнала.
  Снимается через `lockband unlock` — но для этого нужен доступ к роутеру по LAN, а не через
  сам модем. Не делайте это удалённо, если нет запасного канала.
- **Не смешивайте механизмы.** Используйте что-то одно: фиксацию EARFCN, фиксацию PCI, либо
  приоритет бэндов (`AT^BAND_PRI`). Всё вместе — конфликт приоритетов.
- `lockband at` пишет в EFS модема что угодно и **может его окирпичить**. На свой страх и риск.

## Выпуск релиза

Репозиторий публичный, поэтому релизы GitHub публичны сами по себе — ничего,
кроме тега, не требуется:

```sh
git tag v0.1.0
git push origin v0.1.0
```

CI прогонит тесты, соберёт бинари под все архитектуры и создаст релиз с ассетами
`modemui-<target>` и `lockband`.

Перед первым релизом замените заглушки `OWNER/REPO` в [`setup.sh`](setup.sh)
(переменная `REPO`), в этом README и в [`webui/README.md`](webui/README.md).
